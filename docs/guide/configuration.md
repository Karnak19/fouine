# Configuration

fouine reads configuration from environment variables. Some settings (API key, model, prompt) can also be set via the dashboard, which takes precedence over env vars.

## Environment variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `PORT` | no | `3000` | HTTP port |
| `LOG_LEVEL` | no | `info` | Log level: `debug`, `info`, `warn`, `error` |
| `DATA_DIR` | no | `./data` | Directory for SQLite DB, bare repos, worktrees |
| `DB_PATH` | no | `${DATA_DIR}/fouine.db` | SQLite database path |
| `BETTER_AUTH_SECRET` | no‚ÄÝ | ‚Äî | Secret used to sign login sessions (`openssl rand -base64 32`) |
| `BETTER_AUTH_URL` | no‚ÄÝ | `http://localhost:$PORT` | Public origin of the app (OAuth callback base) |
| `ALLOWED_GITHUB_USERS` | no | ‚Äî | Comma-separated GitHub usernames allowed to sign in |
| `GITHUB_APP_ID` | **yes** | ‚Äî | GitHub App ID |
| `GITHUB_APP_PRIVATE_KEY` | yes* | ‚Äî | App private key (literal `\n` are un-escaped) |
| `GITHUB_APP_PRIVATE_KEY_PATH` | yes* | ‚Äî | Path to the `.pem` file |
| `GITHUB_WEBHOOK_SECRET` | **yes** | ‚Äî | Webhook secret for signature verification |
| `GITHUB_APP_CLIENT_ID` | no‚ÄÝ | ‚Äî | The App's OAuth client ID (dashboard login) |
| `GITHUB_APP_CLIENT_SECRET` | no‚ÄÝ | ‚Äî | The App's OAuth client secret |
| `OPENCODE_API_KEY` | recommended | ‚Äî | OpenCode provider API key |
| `OPENCODE_MODEL` | no | `opencode-go/glm-5.2` | Default model for reviews |
| `REVIEW_TIMEOUT_MS` | no | `600000` (10 min) | Max review duration in milliseconds |
| `OPENCODE_CONFIG_DIR` | no | ‚Äî | Path to OpenCode config (tools directory) |

\* Provide the private key via one of the two variables. `*_PATH` is recommended.

‚ÄÝ Login is disabled unless `BETTER_AUTH_SECRET`, `GITHUB_APP_CLIENT_ID` and `GITHUB_APP_CLIENT_SECRET` are all set.

## Dashboard settings

The dashboard (accessible at your server URL) allows setting:

- **OpenCode API key** ‚Äî overrides `OPENCODE_API_KEY`
- **Default model** ‚Äî overrides `OPENCODE_MODEL`
- **Default prompt** ‚Äî the base review prompt used for all repos without a custom prompt

## Per-repo settings

Each registered repo can have:

- **Custom prompt** ‚Äî overrides the default prompt for that repo
- **Custom model** ‚Äî overrides the default model for that repo
- **Enabled/disabled** ‚Äî toggle reviews without removing the repo

## Dashboard login

The dashboard uses **GitHub OAuth**, reusing the **same GitHub App** you already configured for webhooks ‚Äî no separate OAuth App. It is enabled once `BETTER_AUTH_SECRET`, `GITHUB_APP_CLIENT_ID` and `GITHUB_APP_CLIENT_SECRET` are all set; leave any empty for local dev with no login.

In your GitHub App's settings:

1. **General ‚Üí Client ID**: copy it into `GITHUB_APP_CLIENT_ID`. Generate a client secret ‚Üí `GITHUB_APP_CLIENT_SECRET`.
2. **General ‚Üí Callback URL**: add `${BETTER_AUTH_URL}/api/auth/callback/github`.
3. **Permissions ‚Üí Account permissions ‚Üí Email addresses: Read-only**. Required ‚Äî GitHub Apps derive OAuth email from permissions (not scopes), and login needs an email.
4. List who may sign in in `ALLOWED_GITHUB_USERS` (comma-separated GitHub usernames). Anyone not listed is rejected on first sign-in ‚Äî required, since the app is otherwise open to any GitHub account.

When enabled, `/api` requires a session. The webhook endpoint (`/webhook/github`) and `/health` are always exempt, and the login page itself is public.

On **Coolify**, `BETTER_AUTH_SECRET` and `BETTER_AUTH_URL` are auto-generated by the magic vars in `compose.coolify.yml` (`SERVICE_BASE64_64_FOUINE` / `SERVICE_URL_FOUINE`) ‚Äî you only set `GITHUB_APP_CLIENT_ID`, `GITHUB_APP_CLIENT_SECRET` and `ALLOWED_GITHUB_USERS`. Use the app's Coolify domain for the OAuth callback.
